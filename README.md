# **Тестовое задание на позицию Андроид-разработчика**

## **Стэк**
- Kotlin
- Coroutines
- Views
- MVVM
- OOP
- SOLID
- Clean architecture
- Manual DI
- Manual ViewModel Factory
- Retrofit
- Gson
- Glide
- Custom test runner
- Github Actions
- Espresso
- JUnit

---

## **Задача**
Написать погодное Android-приложение на Kotlin.

---

## **Требования**
- Весь интерфейс написать программно, без использования XML.
- Отобразить один экран с погодной информацией:
  - текущая погода
  - почасовая погода (для текущего дня)
  - прогноз погоды на 3 дня
  - город Москва
- Обработать:
  - показ индикатора загрузки
  - стандартный диалог при возникновении ошибки
  - возможность повторить запрос

---

## **UI**
По дизайну никаких ограничений нет — можно ориентироваться на любое стандартное погодное приложение или примеры из интернета и сделать его в упрощённом стиле.
